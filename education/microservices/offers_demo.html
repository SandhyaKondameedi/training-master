Offers Lab

Session Summary

In this lab ssession , we will be covering the following topics
  1. create two microservices ia and offers. Offers is the consumer of ia
  2. Integration with Feign on the offers service
  3. Integration with Ribbon on the offers service
  4. Integration with Eureka pcf service with offer and ia.
  5. Integration with Circuit Breaker pcf service on Offers.
  6. Testing end to end integration , and run a load test viz. postman/jmeter

Feign Integration

  1. In the offers pom , add the following dependency 
                                     
                                            
                                                    org.springframework.cloud
                                                    spring-cloud-starter-openfeign
  2. enable feign clients in the spring boot application, provide the root package where all your classes resides 
     
                                        @SpringBootApplication                                                       
                                        @EnableFeignClients("io.uslab.offers")
                                        public class OffersApplication {
                                        
                                            public static void main(String[] args) {
                                        
                                                SpringApplication.run(OffersApplication.class, args);
                                            }
                                        }
  3. Create a interface IAServiceProxy in the package io.uslab.offers.proxy, url is the end point for local ia service 
                                     
                                    @FeignClient(name="IA",url="https://ia-grouchy-numbat.cfapps.io")
                                    public interface IAServiceProxy {
                                    
                                        @GetMapping("/ia/{pageId}/{placementId}")
                                        public OfferDetails retriveOfferDetails
                                                (@PathVariable("pageId") String pageId,@PathVariable("placementId") String placementId);
                                    }
  4. update the offers controller retriveOffer() to use feign proxy instead of using rest template client 
                                        @RestController
                                    public class OffersController {
                                        @Autowired
                                        private IAServiceProxy proxy;
                                        @GetMapping("/offers")
                                        public OfferDetails retriveOffer(){
                                            OfferDetails response = proxy.retriveOfferDetails("123-43-84","WF_CON_HP_PRIMARY_BNR_1");
                                            return new OfferDetails(response.getPageId(),response.getDescription(),
                                                    response.getPlacementId(),response.getOfferId(),response.getPlacementUrl());
                                        }
                                    }
  5. build and Run the application , and test if offers is able to call ia using feign

Ribbon Integration

  1. Add the following dependency in offers pom 
                                        
                                            org.springframework.cloud
                                            spring-cloud-starter-netflix-ribbon
  2. Add ribbon client to IAServiceProxy interface 
                                        @FeignClient(name="IA")                               
                                    @RibbonClient(name="IA")
                                    public interface IAServiceProxy 
                                        @GetMapping("/ia/{pageId}/{placementId}")
                                        public OfferDetails retriveOfferDetails
                                                (@PathVariable("pageId") String pageId,@PathVariable("placementId") String placementId);
  3. update offers manifes.yml file to add ribbon, list the IA URL 
                                          IA:
                                        ribbon:
                                            listOfServers: localhost:8080
  4. Build and Run offers and test the integration in local envirnoment

Security workaround

  1. Add spring security dependency to offers pom 
                                                
                                                    org.springframework.boot
                                                    spring-boot-starter-security
  2. Create a class OffersConfiguration in the package io.uslab.offers.Configuration 
                                        @Configuration
                                    @EnableWebSecurity
                                    public class OffersConfiguration extends WebSecurityConfigurerAdapter {
                                        @Override
                                        protected void configure(HttpSecurity http) throws Exception{
                                            http.authorizeRequests().anyRequest().permitAll();
                                        }
                                    }
  3. Build Run and Test the services in local envirnoment

Eureka Integration

  1. Add the following dependency in both ia and offers pom 
                                        
                                            io.pivotal.spring.cloud
                                            spring-cloud-services-starter-service-registry
  2. Register and discover through eureka in offers spring boot application 
    
                                                @SpringBootApplication
                                                @EnableEurekaClient
                                                @EnableFeignClients("io.uslab.offers")
                                                @EnableDiscoveryClient
                                                public class OffersApplication {
                                                
                                                    public static void main(String[] args) {
                                                
                                                        SpringApplication.run(OffersApplication.class, args);
                                                    }
                                                }
  3. Update the manifest file , remove ribbon configuration 
                                        applications:
                                    - name: offers
                                      memory: 1G
                                      instances: 1
                                      path: target/offers-0.0.1-SNAPSHOT.jar
                                      random-route: true
                                      services:
                                      - service-registry
  4. Just Build the offers app , don't run it
  5. Register and discover through eureka in ia spring boot application 
        
                                        @SpringBootApplication

                                        @EnableDiscoveryClient

                                        }
  6. Update the manifest file , remove ribbon configuration 
                                                applications:
                                            - name: IA
                                              memory: 1G
                                              instances: 1
                                              path: target/IA-0.0.1-SNAPSHOT.jar
                                              random-route: true
                                              services:
                                              - service-registry
  7. Build the IA projects
  8. In the PCF market place create 2 services Don't bind the service . You can also create it through cf command line refer pcf command line reference.
  9. First deploy the ia service if it's succeed then proceed to offers service , deploy using - cf push ia -p target/ia-0.0.1-SNAPSHOT.jar --random-route (you also hardcode the route in yml file)

Circuit Breaker Integration

  1. Add circuit breaker dependency to offers pom 
                                                
                                                    io.pivotal.spring.cloud
                                                    spring-cloud-services-starter-circuit-breaker
  2. @EnableCircuitBreaker in OffersApplication 
                                                @SpringBootApplication
                                            @EnableEurekaClient
                                            @EnableCircuitBreaker
                                            @EnableFeignClients("io.uslab.offers")
                                            @EnableDiscoveryClient
                                            public class OffersApplication {

                                                public static void main(String[] args) {

                                                    SpringApplication.run(OffersApplication.class, args);
                                                }
                                            }
  3. update OffersController 
                                            @RestController
                                        public class OffersController {
                                        
                                            @Autowired
                                            private IAServiceProxy proxy;
                                        
                                            @GetMapping("/offers")
                                            @HystrixCommand(fallbackMethod ="backUpIAMethod")
                                            public OfferDetails retriveOffer(){
                                                OfferDetails response = proxy.retriveOfferDetails("123-43-84","WF_CON_HP_PRIMARY_BNR_1");
                                                return new OfferDetails(response.getPageId(),response.getDescription(),
                                                        response.getPlacementId(),response.getOfferId(),response.getPlacementUrl());
                                            }
                                            public OfferDetails backUpIAMethod(){
                                        
                                                return new OfferDetails("1345","response from backup services",
                                                        "WF_CON_HP_PRIMARY_BNR_1","C_ent_overdraftrewind_hpprimary_web",
                                                        "https://www20.backup.com/assets/images/contextual/banner/student-loans/1200x532/wfic412_ph_g-187715263_1200x532.jpg");
                                            }
                                        }
  4. update the manifest file of offers with circuit breaker service 
                                        applications:
                                    - name: offers
                                      memory: 1G
                                      instances: 1
                                      path: target/offers-0.0.1-SNAPSHOT.jar
                                      random-route: true
                                      services:
                                      - service-registry
                                      - circuit-breaker
                                    
                                    eureka:
                                      client:
                                        serviceUrl:
                                          defaultZone: https://eureka-fc919389-9031-4eb0-8e7f-91bc132aac57.cfapps.io
  5. Build the service
  6. deploy the service - cf push ia -p target/offers-0.0.1-SNAPSHOT.jar --random-route
  7. bind the service - cf bind-service offers circuit-breaker
  8. cf restart offers
  9. test the offers hitting offers endpoint
  10. In PCF navigate to offer's circuit breaker click manage and check circuit breaker dash board
  11. Install postman and run a negative scenario like for circuit close scenario's
